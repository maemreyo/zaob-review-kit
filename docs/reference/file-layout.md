# File Layout After Install

What your filesystem looks like after running `zrk install-all` (example: Kiro).

---

## Global files (per machine, all projects)

```
~/.kiro/steering/
  review-roles.md          ← 5 reviewer role definitions
  review-prompting.md      ← output structure and calibration
  review-ignore.md         ← default ignore patterns
  review-memory.md         ← review memory format
  .zrk-manifest.json       ← installed file hashes (powers zrk status)
```

For Claude Code:

```
~/.claude/commands/review-kit/
  review-roles.md
  review-prompting.md
  review-ignore.md
  review-memory.md
  .zrk-manifest.json
```

---

## Workspace files (per project)

```
your-project/
  .kiro/steering/
    prep-review.md          ← main workflow trigger
    pack-materials.md       ← context XML packing rules
    project-context.md      ← fill this in yourself
    .zrk-manifest.json
```

For all four agents at once (`--all-targets`):

```
your-project/
  .kiro/steering/
    prep-review.md
    pack-materials.md
    project-context.md
    .zrk-manifest.json
  .claude/commands/review-kit/
    prep-review.md
    pack-materials.md
    project-context.md
    .zrk-manifest.json
  .cursor/rules/
    prep-review.mdc
    pack-materials.mdc
    project-context.mdc
    .zrk-manifest.json
  .windsurf/rules/
    prep-review.md
    pack-materials.md
    project-context.md
    .zrk-manifest.json
```

## Template files (project root)

```
your-project/
  .archignore              ← project-specific ignore patterns (customize this)
  .gitignore               ← zrk appends a snippet to this file
```

---

## Review materials (generated by your agent)

```
your-project/
  .materials/
    review_memory.md       ← commit this
    REVIEW_LOG.md          ← commit this
    20250304_a1b2c3/       ← gitignored
      review_context.xml
      review_prompt.md
      review.patch
    20250310_d4e5f6/       ← gitignored
      ...
```

---

## What to commit

| Path                          | Commit? | Reason                            |
| ----------------------------- | ------- | --------------------------------- |
| `.kiro/steering/*.md`         | Yes     | Agent config, part of the project |
| `.archignore`                 | Yes     | Part of the project config        |
| `.materials/review_memory.md` | Yes     | Shared project knowledge          |
| `.materials/REVIEW_LOG.md`    | Yes     | History                           |
| `.materials/<timestamp>/`     | No      | Too large, regenerate as needed   |
| `.zrk-manifest.json`          | No      | Machine-local metadata            |

The `.gitignore` snippet added by zrk handles the gitignored entries automatically.
